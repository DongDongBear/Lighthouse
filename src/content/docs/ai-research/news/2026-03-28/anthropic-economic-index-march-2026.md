## 文章背景

2025 年 9 月，Anthropic 发布了首期经济指数报告，首次尝试用其自有平台数据系统性地回答"AI 如何影响经济"这一问题。该报告建立了基线方法论：通过对 Claude.ai 和第一方 API 的对话抽样，将每一次对话映射到美国劳工统计局（BLS）的职业-任务框架中，从而量化 AI 对不同工作的渗透程度。

六个月后的第二期报告——副标题为"Learning Curves"（学习曲线）——在方法论不变的前提下，追踪了关键指标的时间演变。数据采集窗口为 2026 年 2 月 5-12 日，恰好与 Opus 4.6 模型发布同期。样本量为 Claude.ai 和第一方 API 上的 100 万次对话，采用 O*NET 2019 职业框架和 BLS 2024 年 5 月工资数据，全程遵循隐私保护分析协议。

这份报告的核心关切不再是"AI 被用来做什么"，而是**"AI 的使用模式如何随时间演变，这种演变对经济意味着什么"**。

## 完整内容还原

### 一、使用场景正在快速多元化

报告最醒目的趋势之一是 **任务集中度的持续下降**。上期报告中，排名前 10 的任务占总流量的 24%；本期这个数字降至 19%。这意味着用户正在将 Claude 应用到越来越多样化的工作场景中，AI 的经济渗透正在从少数高频用例向长尾扩散。

具体来看，几个重要的结构性转变正在发生：

- **课业用途大幅下降：** 从 19% 降至 12%。这与直觉一致——当学生群体的早期采纳红利消退后，AI 的用户结构正在向职场工作者倾斜。
- **个人用途显著上升：** 从 35% 上升至 42%。越来越多的人将 AI 融入日常生活而非仅仅是工作或学习。
- **职业渗透率达到新高：** 49% 的职业已经有 25% 以上的任务在 Claude 上完成。接近一半的职业类别已经实质性地与 AI 交织在一起。

### 二、编程使用的平台迁移

一个值得关注的平台动态是 **编程任务从 Claude.ai 向 API 迁移**。这主要受 Claude Code 等开发工具的驱动——程序员越来越多地通过 API 集成而非网页界面来使用 AI 编程能力。

这一迁移在模型选择偏好上也有体现：Opus 在编码任务上的使用份额增加了 4 个百分点，而在辅导/教学（tutoring）任务上则下降了 7 个百分点。高复杂度的编程工作倾向于选择最强模型，而教学辅导等对话类任务对模型能力的要求相对更宽容。

### 三、平均任务价值的微妙变化

报告估算的对话平均任务价值从上期的 $49.30 微降至 $47.90。表面看这似乎意味着 AI 的经济产出在"贬值"，但报告的解释更为审慎：这反映的是 **采纳曲线向更广泛受众扩散的自然结果**。

早期用户往往是高技能、高薪职业的从业者，他们使用 AI 处理的任务本身价值较高。随着 AI 渗透到更广泛的人群中，平均任务价值的轻微下降恰恰说明 AI 正在从精英工具走向大众工具——这是健康的扩散信号，而非价值衰减。

### 四、收入与模型选择的精确对应

报告揭示了一个极其精确的经济学发现：**每增加 $10 的时薪，用户选择 Opus 模型的概率在 Claude.ai 上增加 1.5 个百分点，在 API 上增加 2.8 个百分点。**

这一发现至少有两层含义。首先，高收入用户对模型能力的边际需求更高——他们处理的任务更复杂，愿意为更强的模型"付费"（通过订阅或 API 成本）。其次，API 端的斜率（2.8pp）几乎是 Claude.ai 端（1.5pp）的两倍，这暗示 API 用户群体中收入-需求的关联更为紧密，可能因为 API 使用本身就预设了更高的技术门槛和更明确的商业目的。

用户受教育年限的平均值从 12.2 年略降至 11.9 年，与任务价值下降的趋势一致，进一步佐证了"AI 正在向更广泛人群扩散"的判断。

### 五、核心发现——学习效应（Learning-by-Doing）

本报告最重要的发现以"Learning Curves"为题直接点明：**使用 Claude 6 个月以上的高任期用户，在完全控制变量后，仍然比新用户展现出约 4 个百分点的更高任务成功率。**

"完全控制变量"意味着研究者排除了以下干扰因素：

- 用户的职业、教育背景、收入水平
- 任务类型、复杂度、领域
- 模型版本（Opus / Sonnet / Haiku）
- 平台（Claude.ai / API）

即便如此，长期使用经验本身仍然带来了可测量的绩效提升。这是一个纯粹的"学习效应"——人们确实在与 AI 的反复交互中变得更擅长使用它。

高任期用户的行为模式也与新用户存在显著差异：

- **更多工作相关用途：** 高任期用户的工作相关对话比例比新用户高 7 个百分点。换言之，长期用户逐渐将 AI 从"尝鲜玩具"转化为"生产力工具"。
- **更高的迭代性和协作性：** 长期用户更倾向于多轮对话、反复修改、逐步精化结果，而非一次性查询。
- **更多教育类高级任务：** 长期用户参与更复杂的教育和知识构建活动，而非简单的问答。

### 六、增强 vs 自动化的天平

报告延续了上期对"AI 增强"（augmentation，人机协作）与"AI 自动化"（automation，AI 独立完成）的区分，并发现了分化趋势：

- **Claude.ai 端：** 增强模式占比在上升。用户越来越多地将 Claude 作为思维伙伴而非代劳工具使用。
- **API 端：** 自动化模式正在快速增长。销售外联自动化（sales/outreach automation）和自动交易（automated trading）等 API 自动化模式的规模翻倍。

这形成了一个清晰的双轨格局：消费者端（Claude.ai）走向人机协作，开发者端（API）走向全自动化。两者并非矛盾，而是 AI 在不同使用场景中的自然分化。

### 七、地理格局：国内收敛，全球发散

报告在地理维度上发现了一个看似矛盾但实则深刻的趋势：

**美国国内：** 各州之间的 AI 使用率正在收敛。按照当前趋势，落后州和领先州之间的差距预计需要 5-9 年才能基本抹平。这说明 AI 在美国国内的扩散遵循经典的技术扩散S曲线——领先地区增长放缓，落后地区加速追赶。

**全球层面：** 差距却在扩大。排名前 20 的国家在全球 AI 使用中的份额从 45% 上升至 48%。这意味着发达国家与发展中国家之间的"AI 鸿沟"非但没有缩小，反而在加深。语言壁垒、基础设施差距、支付能力差异和互联网接入质量可能都是推动这一发散的因素。

## 核心技术洞察

### 洞察一：学习效应证明了"AI 素养"是一种可积累的人力资本

4 个百分点的成功率差异在控制了所有可观察变量之后仍然存在，这在经济学上等价于说：**AI 使用技能是一种独立的人力资本形式。** 它不能被教育水平、职业经验或任务复杂度所替代——它必须通过实际使用来习得。

这一发现的政策含义深远。如果 AI 素养是需要时间积累的人力资本，那么早期接触 AI 的群体将获得复利式的能力优势。企业层面，这意味着越早系统性地将 AI 融入工作流的组织，其员工将积累越多的"AI 人力资本"，这种优势会随时间加速扩大。

### 洞察二：平台分化暗示了 AI 经济的"两个世界"

Claude.ai 上增强模式增长、API 上自动化模式翻倍——这不仅仅是技术偏好的差异，它预示着 AI 经济可能沿着两条截然不同的路径演进：

- **增强路径：** 人类保持在决策环中，AI 提升人类的生产力和决策质量。劳动力市场的影响是工资溢价和技能重组，但就业总量变化不大。
- **自动化路径：** AI 独立执行完整的工作流程（如自动销售外联、自动交易）。劳动力市场的影响是特定岗位的直接替代。

两条路径同时加速，意味着"AI 对就业是好是坏"这个问题没有统一答案——取决于你在哪条路径上。

### 洞察三：$10 时薪 → Opus 使用率的精确弹性揭示了"AI 模型即生产资料"

每 $10 时薪对应 1.5-2.8pp 的 Opus 选择倾向，这个数字之所以重要，是因为它证明了 **AI 模型的能力层级正在被市场精确定价**。高收入用户（处理高价值任务的人）系统性地选择更强的模型，就像高产出的工厂购买更高规格的机床一样。AI 模型不再是一个"免费的智能助手"，它正在变成一种有明确产出弹性的生产资料。

## 实践指南

### 对企业决策者

1. **立即启动 AI 技能积累计划。** 学习效应是有时间复利的——等待 6 个月再开始意味着竞争对手的员工已经拥有了可测量的能力优势。不要等"完美的 AI 战略"出台后再行动，先让员工开始使用。

2. **根据任务价值匹配模型层级。** $10 时薪对 Opus 使用率的弹性数据提供了一个实用的框架：高价值任务（时薪 $50+）应分配最强模型，低价值任务可以使用更经济的模型。这不是直觉，是数据。

3. **关注 API 自动化的翻倍趋势。** 如果你的企业涉及销售外联、客户开发或金融交易等领域，竞争对手很可能已经在使用 API 自动化这些流程。自动化模式翻倍意味着这已经从"实验"进入了"规模化"阶段。

### 对政策制定者

1. **AI 素养的"数字鸿沟"可能比互联网时代更严重。** 全球排名前 20 国家的 AI 使用份额从 45% 升至 48%。与互联网不同，AI 的有效使用需要更高的语言能力和认知技能门槛，这使得追赶更加困难。

2. **美国国内 5-9 年的收敛周期是一个重要的政策窗口。** 如果任由市场自然扩散，落后地区需要近十年才能追赶。有针对性的干预（如 AI 培训补贴、公共部门 AI 采纳计划）可能显著缩短这一周期。

3. **课业用途从 19% 降至 12% 需要重新审视教育政策。** 这不一定意味着学生用 AI 变少了——更可能是学生在总用户中的占比被职场用户稀释了。但教育系统仍需要主动引导学生从"用 AI 做作业"转向"用 AI 学习如何学习"。

### 对个人用户

1. **持续使用比间歇使用更有价值。** 学习效应需要时间积累。6 个月以上的持续使用才能带来统计上可测量的能力提升。将 AI 使用视为一种需要"练习"的技能，而非一个即插即用的工具。

2. **有意识地从"问答"模式转向"协作"模式。** 高任期用户的核心行为特征是迭代性和协作性——他们不是向 AI 提一个问题等一个答案，而是通过多轮对话逐步精化结果。这种使用方式的学习回报更高。

## 批判性分析

### 方法论层面的局限

**选择偏差问题。** 报告的数据仅来自 Claude 平台——这意味着它无法代表整个 AI 市场。使用 ChatGPT、Gemini、开源模型的用户可能有完全不同的使用模式和经济特征。Anthropic 的用户群体可能系统性地偏向更技术导向、更高收入的人群（尤其是 API 端），这会扭曲对"AI 经济影响"的整体估计。

**学习效应的因果推断。** 虽然报告声称"完全控制变量后"仍有 4pp 的成功率差异，但严格来说，这仍然是观察性研究而非随机实验。存在无法观测的混淆变量的可能性——例如，"坚持使用 AI 6 个月以上的人"本身可能就是学习能力更强、适应性更高的人群。即使控制了教育和职业，这种"未观测的能力偏差"仍可能部分解释所观察到的学习效应。

**O*NET 2019 框架的滞后性。** 报告使用的职业-任务分类体系基于 2019 年版 O*NET，这意味着 AI 本身催生的新职业（如 prompt engineer、AI agent 开发者）和 AI 消灭的旧任务可能无法被准确映射。随着 AI 对劳动市场的重塑加速，这一框架的适用性将持续衰减。

### 数据解读层面的警示

**Opus 4.6 发布的混淆效应。** 数据采集窗口（2026 年 2 月 5-12 日）恰好与 Opus 4.6 发布重叠。新模型发布通常会引发使用模式的短期波动——好奇心驱动的尝鲜、从旧模型的迁移、对新能力的探索性使用。报告中观察到的某些趋势（如 Opus 使用份额变化、编程任务向 API 迁移）可能部分是这种短期波动的结果，而非稳态趋势。

**任务价值估算的粗糙性。** $47.90 的"平均任务价值"是通过将对话映射到 BLS 工资数据后估算的——这假设用户完成一个 AI 辅助任务所"节省"的价值等于该任务在人工市场上的价格。但现实中，AI 辅助任务的价值可能远高于或远低于其人工替代成本，取决于质量差异、速度溢价和创新价值。

### 缺失的维度

**企业级使用完全缺席。** 报告仅覆盖 Claude.ai（消费者端）和第一方 API。通过 AWS Bedrock、Google Cloud Vertex 等第三方渠道接入 Claude 的企业用户——可能是经济产出最大的群体——完全不在分析范围内。这意味着报告可能系统性地低估了 AI 的经济渗透程度。

**质量维度的忽视。** 报告关注"AI 被用来做什么"和"使用频率如何"，但几乎没有触及"AI 做得有多好"。成功率指标虽然存在，但缺乏与人类基准的系统性对比。一个任务被"完成"并不等于它被"高质量地完成"。

**竞争生态的盲区。** 报告隐含地假设 Claude 使用趋势可以代表整个 AI 市场。但不同平台之间可能存在显著的用户迁移——例如，Claude 上编程任务的"减少"是否只是转移到了 Cursor、GitHub Copilot 等专用工具？没有跨平台数据，单一平台的趋势可能产生误导性结论。

### 独立观察

这份报告最深远的意义可能不在于任何单个数据点，而在于它建立了一个 **可纵向追踪的 AI 经济指标体系**。在此之前，关于"AI 对经济的影响"的讨论充斥着预测和猜想；现在，Anthropic 每半年更新一次的经济指数提供了真实的、有时间序列的经验证据。

但这也引发了一个值得警惕的问题：**当评估 AI 经济影响的数据完全来自 AI 公司本身时，我们应该如何看待它的客观性？** Anthropic 既是被评估对象又是评估者——即使方法论是公开和严谨的，选择展示哪些数据、如何框定叙事（"学习曲线"暗示的积极进步感），仍然是公司的编辑决策。独立的第三方验证——由政府统计机构或学术研究者使用类似方法但不同数据源进行——将是这一指标体系获得公信力的关键下一步。

此外，全球使用率的发散趋势（Top 20 国家从 45% 升至 48%）与学习效应的结合，可能指向一个令人不安的循环：已经在使用 AI 的群体持续积累"AI 人力资本"，尚未接触 AI 的群体在差距扩大中更难追赶。如果这种"AI 马太效应"真实存在，它对全球不平等的影响可能远超互联网时代——因为 AI 的生产力放大系数远大于简单的信息获取。
