## 文章背景

### 为什么是现在？

2026 年 4 月 10 日是一个值得记录的日期。就在 CoreWeave 同日宣布与 Anthropic 和 Meta 签署超百亿算力合约、Stargate UK 暂停的新闻充斥媒体时，Anthropic 悄悄做了一件在 AI 史上尚属首次的事：正式承认他们有一个"太危险"而不能公开发布的模型，但同时以定向封闭的方式将其落地为商业产品。

Project Glasswing 的时间节点选择并非偶然：

- Anthropic IPO 预期在 2026 年，需要向机构投资者展示"负责任 AI"的差异化定位
- 大模型在网络安全领域的应用竞争已进入白热化（Google Sec-Palm、OpenAI Cyber 产品线）
- 开源软件供应链攻击事件频发（XZ Utils 后门、SolarWinds 后继影响），防御侧的 AI 自动化需求明确
- Claude Opus 4.6 已经在 CyberGym 达到 66.6%，为 Mythos 的更高性能提供了足够的基础

这篇博文不是一个单纯的技术发布，而是 Anthropic 对"高风险 AI 能力如何负责任商业化"这一命题给出的第一个完整答案。

### Glasswing 的命名哲学

项目名来自透翅蝶（*Greta oto*，又称玻璃翼蝴蝶）。这个命名是精心设计的双重隐喻：

1. **透明性**：玻璃翼蝴蝶的翅膀几乎完全透明，象征着 Anthropic 倡导的安全透明度——漏洞应该被找到、公开、修复，而不是隐藏在代码深处。

2. **伪装中的危险**：正是因为翅膀透明，玻璃翼蝴蝶能在植被中完美隐身——就像潜伏在软件中 27 年的 OpenBSD 漏洞，被数百万次自动化测试扫过却从未被发现。

## 完整内容还原

### Claude Mythos 是什么？

Claude Mythos Preview 是 Anthropic 构建的第一个专为网络安全防御设计的专项模型，而非通用 AI 助手的改良版本。它是专门针对以下核心任务设计和微调的：

- **漏洞发现**：自主阅读代码库，识别潜在安全漏洞
- **漏洞复现**：给出已知漏洞的可验证利用路径（Proof-of-Concept）
- **代码审计**：大规模扫描开源代码库的安全缺陷
- **渗透测试辅助**：协助安全研究人员的红队工作

### 基准测试：数字说话

Mythos Preview 与 Claude Opus 4.6（当前最强通用模型）的正面对比：

| 基准测试 | Claude Mythos Preview | Claude Opus 4.6 | 提升幅度 |
|---|---|---|---|
| **CyberGym**（漏洞复现） | **83.1%** | 66.6% | +24.8% |
| **SWE-bench Verified** | **93.9%** | 80.8% | +16.2% |
| **SWE-bench Pro** | **77.8%** | 53.4% | +45.7% |
| **Terminal-Bench 2.0** | **82.0%** | 65.4% | +25.4% |
| **SWE-bench Multilingual** | **87.3%** | 77.8% | +12.2% |
| **GPQA Diamond** | **94.6%** | 91.3% | +3.6% |
| **HLE（含工具）** | **64.7%** | 53.1% | +21.8% |

**解读这些数字的正确方式：**

最重要的是 **CyberGym** 和 **SWE-bench Pro** 这两项，因为它们直接衡量安全相关能力：

- **CyberGym 83.1%**：这是一个衡量"能否自主完成真实漏洞复现任务"的基准，而非合成题目。从 66.6% 到 83.1%，提升了约 25%。在安全研究中，这意味着覆盖的漏洞类型和自动化深度有根本性变化——能做的任务种类更多，更难的漏洞也能处理。

- **SWE-bench Pro 45.7% 的相对提升**：SWE-bench Pro 比 Verified 版本更难，代表更接近真实工程任务的代码修复能力。从 53.4% 到 77.8% 是一次量级跳跃。

- **GPQA Diamond 94.6%**：接近满分，说明 Mythos 在专业科学推理上也没有损失——这是通用能力保留的证明，不是狭窄专项牺牲了通用性。

### 已发现的真实漏洞

Mythos Preview 在测试阶段发现的三类真实零日漏洞（全部经过负责任披露和修复）：

#### 1. OpenBSD 27 年历史的远程崩溃漏洞

OpenBSD 是以安全性著称的操作系统——整个操作系统的设计哲学就是"安全优先"，用于服务器、防火墙和关键基础设施。一个存在了 27 年、允许远程触发崩溃的漏洞，意味着在此之前所有的人工审计和自动化扫描都没有找到它。

Mythos 能找到它的原因很可能是：它能在大规模代码库中同时跟踪多个执行路径和状态假设，而这超出了人类审计者的工作记忆容量，也超出了传统静态分析工具的符号推理边界。

#### 2. FFmpeg 16 年历史漏洞（五百万次自动化测试未命中）

FFmpeg 是全球使用最广泛的多媒体处理库，存在于几乎所有播放器、视频平台和转码服务中。这个漏洞被自动化测试工具扫描了**五百万次**而没有触发，说明它需要非常具体的输入条件才能被激活——这恰恰是传统模糊测试的盲区，而 Mythos 的代码推理能力可以"从逻辑上推导"出触发路径，而不依赖随机覆盖。

#### 3. Linux 内核权限提升漏洞链

多个独立漏洞被链式组合，从普通用户权限提升到完全系统控制。这种"漏洞链"发现的难度远超单一漏洞，因为需要同时理解多个子系统的交互和边界条件。

**关键细节：** Anthropic 强调，Mythos "**完全自主地**识别了这些漏洞，没有任何人工引导"。这不是"AI 辅助人类发现漏洞"，而是"AI 独立完成了漏洞发现的完整工作流"。

### 定价结构与访问层级

**Mythos Preview 定价：$25/$125（输入/输出，每百万 token）**

对比 Claude 当前通用 API 的定价层级（Claude Haiku 约 $0.25/$1.25，Claude Opus 约 $15/$75），Mythos 的定价约是 Opus 的 **1.7 倍**，是整个产品线中的绝对高端。Fortune 报道指出，Mythos "比 Opus 大很多倍，也贵很多倍"——这直接解释了为什么不能大规模开放：服务成本极高。

**访问层级：**

1. **第一层：12 家创始合作伙伴**（直接访问权限）
   - 云计算：AWS
   - 硬件与设备：Apple、Broadcom、NVIDIA
   - 网络与安全：Cisco、CrowdStrike、Palo Alto Networks
   - 金融服务：JPMorganChase
   - 通用科技：Google、Microsoft
   - 基础设施治理：Linux Foundation

2. **第二层：40+ 家关键基础设施维护者**（经审核访问）

3. **第三层：未来 Cyber 验证项目**（面向安全专业人员的申请通道）

**$100M 信用额度**：Anthropic 为 Project Glasswing 参与者提供 $1 亿的使用信用，这是 Anthropic 罕见的大规模前期投入，直接覆盖了 12 家伙伴前期使用的成本壁垒。

### 资助生态系统

Anthropic 同时宣布向开源基础设施安全生态捐款：

- **$2.5M → Alpha-Omega + OpenSSF**（通过 Linux Foundation）
  - Alpha-Omega：专注于开源软件供应链安全，主要资助高影响力开源项目的安全审计
  - OpenSSF（Open Source Security Foundation）：Linux Foundation 旗下，制定开源安全标准和最佳实践

- **$1.5M → Apache Software Foundation**
  - Apache 托管数百个全球广泛使用的开源项目（Kafka、Spark、HTTP Server 等），均是 Mythos 未来可应用的审计对象

**战略逻辑：** 通过捐赠让开源维护者有资源接入 Mythos，实现漏洞发现能力的"自下而上"渗透——不只服务顶级企业，也让全球开源基础设施受益。

### 双重用途的哲学考量

Anthropic 在博文中直接正面回应了"为什么不公开"的道德争议：

> "使 AI 模型在错误的人手中危险的同一能力，使它们在发现和修复缺陷上无价。"

这是一个关于 AI 能力边界的深刻问题，而 Anthropic 的回答是**结构性访问控制**，而非能力削减：

1. **不削弱模型能力**：Mythos 完整地保留了漏洞利用推理能力，不进行针对性 unlearning
2. **通过访问控制实现选择性开放**：谁能用、用在什么场景，由 Anthropic 的合规层管理
3. **护栏检测系统**：Anthropic 正在为 Claude Opus 系列开发专项安全过滤器，计划作为 Mythos 后续版本的配套安全机制

**Cyber 验证项目**是这个哲学的具体落地：安全专业人员可以申请验证身份后获得受限访问权限——类似于化学品管制体系中的"许可证"模式，而非简单的禁止或开放二选一。

## 核心技术洞察

### 1. 专项微调 vs. 通用能力提升

Mythos 的性能飞跃（CyberGym +25%）说明了 AI 能力的一个重要规律：**专项任务的极限性能远高于通用模型的性能上界**，但代价是更高的推理成本和更强的访问限制需求。

这与人类专家的逻辑完全相同：一个 10 年安全专家和一个全栈工程师，在代码审计任务上的差距就是 25%+。AI 的专项微调复现了这种分工。

### 2. "完全自主"是关键门槛

普通 AI 辅助安全工具（如 GitHub Copilot Security Scan）是"建议工具"——人类做决策，AI 提供候选。Mythos 是"自主代理"——给定代码库，自主完成漏洞发现的全流程。

这个差异决定了它能否真正替代（或倍增）安全研究员的工作效率，而不只是"让他们打字更快"。五百万次自动化测试未命中的 FFmpeg 漏洞，Mythos 能找到，就是这个"自主推理"能力的直接体现。

### 3. 定向发布作为新的商业化范式

"太危险但定向可用"是一个全新的产品范式，与以往的"要么公开要么保密"形成对比。

Anthropic 用 Project Glasswing 定义了一个三层访问结构：顶层合作伙伴（直接访问）→ 审核访问者（申请制）→ 未来大众（配套护栏成熟后开放）。这种渐进式开放的模板，可能成为高风险 AI 能力商业化的行业标准。

### 4. 开源安全是杠杆点

Anthropic 的 $4M 开源捐款不是公关，而是产品战略：让 Mythos 渗透进 Apache、OpenSSF 维护的项目，等于间接保障了这些项目下游的所有用户（数十亿设备和服务）。单位杠杆比直接服务企业客户高出数倍。

### 🟢 对安全研究人员（立即相关）

1. **申请 Cyber 验证项目**：Anthropic 承诺开放申请通道，如果你的工作涉及合法漏洞研究，这是目前最快进入 Mythos 访问层级的路径。

2. **关注 OpenSSF/Alpha-Omega 的 Mythos 集成**：$2.5M 的捐赠会推动这两个组织将 Mythos 纳入其安全审计工作流，开源项目维护者可能可以申请访问。

3. **基准对照**：CyberGym 83.1% 是目前已知最强的漏洞复现能力，将其作为"AI 辅助漏洞研究"的性能上界参考，评估自己现有工具链的差距。

### 🟡 对 CISO 和安全负责人（需评估）

1. **CoreWeave/AWS/Google/Microsoft 渠道**：12 家合作伙伴中有多个云服务商，Mythos 很可能会通过这些平台向企业客户提供受限访问，关注各自 Marketplace 动态。

2. **成本结构评估**：$25/$125 的定价意味着一个中等规模渗透测试项目的 AI 使用成本可能在 $5,000-$50,000 区间，与人工安全审计费用相比具有竞争力但需要场景匹配。

### 🔴 注意事项与边界

1. **访问控制的有效性问题**：12 家合作伙伴中同时包含 Google 和 Microsoft（Anthropic 的竞争对手之一），以及直接使用安全服务的 CrowdStrike 和 Palo Alto Networks（Anthropic 的间接竞争对手）。信息共享边界如何划定，是真实的合规挑战。

2. **"完全自主"的可靠性尚未第三方验证**：83.1% 的 CyberGym 成绩是 Anthropic 内部测试结果，等待独立安全研究机构的复现和验证。

3. **漏洞利用 vs. 漏洞发现**：Mythos 能找漏洞，也因此能被用于攻击——这是双重用途的根本矛盾，访问控制机制一旦出现泄露或被绕过，后果不可控。

## 批判性分析

### 被忽略的问题

**1. 合作伙伴内部的信息隔离怎么做？**

Apple 的安全团队拿到 Mythos，Cisco 的安全团队也拿到 Mythos，他们可以用它审计对方的代码吗？12 家合作伙伴使用范围的合同边界是什么？这在 B2B 安全工具历史上是前所未有的复杂场景。

**2. "太危险"的具体威胁模型是什么？**

博文和 Fortune 报道都没有明说 Mythos "危险在哪"。是因为能自动生成漏洞利用代码（exploits）？还是因为它能在不加引导下发现 0-day？如果是前者，那边界应该在输出过滤层面；如果是后者，那访问控制根本无法阻止——因为找到漏洞就等于找到了利用路径。

**3. "完全自主"和"五百万次测试未命中"是否存在过度宣传？**

这两个数字都是 Anthropic 自述，缺乏独立验证。在 AI 能力宣称中，"完全自主"往往意味着"在特定测试环境下没有观察到人工干预"，而非真正意义上的无需任何引导。等待 CrowdStrike/Palo Alto 的独立使用报告。

**4. IPO 前的时机选择**

Anthropic 在 IPO 预期年份的 Q1 发布这一产品，时机不可能是巧合。"负责任 AI"的差异化定位在投资者路演中有直接价值。我们无法判断这是战略决策驱动了产品，还是产品就绪后选择了合适时机。
