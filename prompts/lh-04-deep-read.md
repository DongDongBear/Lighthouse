**"追踪 + 去重"是一体两面。** 深度解读轮的追踪逻辑是：
- **你的"追踪资产"不只是 daily.md 的下期追踪问题，还包括过去 14 天自己写过的每一篇 `deep-*.md`**。每篇深度解读本质上都是一个"正在进行的故事"，后续几天如果有新数字/新实战/新争议，都应该追加到**原文件底部**，而不是重写一篇。

```bash
for d in $(seq 1 14); do
```

### Step 1：每个候选主题必须先 grep 过去 14 天 deep 文件
```bash
for d in $(seq 1 14); do
  date_check=$(date -d "${TODAY} -${d} days" '+%Y-%m-%d')
  ls /tmp/Lighthouse/src/content/docs/ai-research/news/${date_check}/deep-*.md 2>/dev/null | while read f; do
    if grep -l -i -E "<关键词>" "$f" > /dev/null 2>&1; then
      echo "HIT: $f"
    fi
  done
done
```

### Step 2：命中时走三条路径之一

**路径 1｜完全相同事件 + 没有任何新信息** → **不写新解读，也不追加**。跳过这个主题。

**路径 2｜完全相同事件 + 有新披露**（新数字、新合作伙伴、新定价、用户反馈、第三方验证、监管回应）→ **最优解**：在原 `deep-*.md` 文件**底部追加** `## 📌 ${TODAY} 更新` 章节。格式：
```markdown

---

## 📌 ${TODAY} 更新

- **<新披露点 1>**：<具体数字/事实>
- **<新披露点 2>**：<具体数字/事实>
- **评价修正**：<如果新数据改变了原文的某个判断，明确说出来>
```
**不**要在当天的目录下新建 `deep-*.md` 文件。

**路径 3｜同主题但角度完全不同**（例如原文只写了架构，今天有独立的第三方实战评测 / benchmark 对比 / 争议性反驳论文）→ 可以写新文件，但：
- 标题必须明确 "XXX 后续评测"/"XXX 实战验证"/"XXX 争议反驳"
- 文章开头第一段必须 link 到原 deep 文章，并明确"本文不重复原文背景，聚焦 <新角度>"
- 不要重复原文已分析过的技术细节

### 写完后自检
- 对今天生成的每个 `deep-*.md` 标题关键词，grep 过去 14 天的 deep 文件，任何命中都要复审是应该撤回还是改为追加章节
- 如果追加了 `## 📌 更新` 章节，git commit 时要提到"append update to original deep article"，不要写成新文章

```bash
cd /tmp/Lighthouse && \
```

**部署后强校验（必须执行，任何一条失败都算本轮未完成）：**

```bash
TODAY=$(date -d '+8 hours' '+%Y-%m-%d')

# 1) dist 中必须存在今日日报页
test -f "/tmp/Lighthouse/dist/ai-research/news/${TODAY}/daily/index.html"

# 2) nginx 部署目录中必须存在今日日报页
test -f "/usr/share/nginx/html/Lighthouse/ai-research/news/${TODAY}/daily/index.html"

# 3) 如果今天写了 deep-* 解读，nginx 目录中也必须能找到对应页面
find "/tmp/Lighthouse/src/content/docs/ai-research/news/${TODAY}" -maxdepth 1 -name 'deep-*.md' -print | while read -r f; do
  slug=$(basename "$f" .md)
  test -f "/usr/share/nginx/html/Lighthouse/ai-research/news/${TODAY}/${slug}/index.html"
done
```

**汇报规则追加：**
- 回复里必须单列 `DEPLOY_VERIFY` 小节
- 明确写出：
  - dist 校验是否通过
  - nginx 校验是否通过
  - 今日已部署页面清单（至少 daily + 全部 deep 页面）
- 如果没有通过，不得说“已部署成功”

**如果 build 失败：**
1. 检查 frontmatter 格式（最常见问题：中文引号嵌套、缺少 title/description）
2. 修复问题文件
3. 重新 build
4. 如果是 astro 版本/依赖问题：`cd /tmp/Lighthouse/astro-app && npm install && npx astro build`

**如果 git push 失败：**
1. git pull --rebase
2. 解决冲突
3. git push
4. 最后手段：git push --force-with-lease
