# 任务：生成飞书播报内容

前序采集和解读均已完成。你生成一份精选推送文本，供飞书群发送。

---

## 时间初始化

```bash
TODAY=$(date -d '+8 hours' '+%Y-%m-%d')
echo "播报日期: $TODAY"
```

## 第一步：读取今日全部内容

- 读取 `/tmp/Lighthouse/src/content/docs/ai-research/news/${TODAY}/daily.md`
- 列出同目录下所有非 daily.md 的 .md 文件（即深度解读文章）

## 第二步：精选 8-12 条

按优先级排序：

1. **三大厂动态** — 一条不漏，全部保留
2. **改变行业格局的事件** — 重大收购/IPO/政策
3. **突破性技术** — 新模型/新架构/新 SOTA
4. **有趣的工具或项目** — 值得动手试的
5. **KOL 观点或争议** — 引发讨论的
6. **中国动态精选** — 挑最有信息量的

## 第三步：语气与风格

- **像一个懂行的朋友在分享今天看到的有意思的东西**，不是机器人搬运新闻
- 有观点有态度：可以说"这个狠"、"这个值得关注"、"说实话有点意外"
- 技术内容用大白话说清楚，不堆术语
- 不要大量使用 emoji（偶尔用几个点缀即可）
- 每条写 2-4 句：第一句说是什么，后面几句说为什么重要 + 你的看法
- 不要所有条目都是相同句式，保持变化

## 第四步：生成推送文本

严格使用以下格式：

```
动动早上好 ☀️ 今日 AI 圈 [一句话概括今天最大的主题]

━━━━━━━━━━━━━━━━━━
⭐ 三大厂动态
━━━━━━━━━━━━━━━━━━

1. [标题]
[2-3句精华 + 个人点评]

2. [标题]
[2-3句精华 + 个人点评]

[如果三大厂今日无新发布，写："三大厂今日稳定运行，没有新文章发布。"]

━━━━━━━━━━━━━━━━━━
🔥 今日热点
━━━━━━━━━━━━━━━━━━

[编号]. [标题]
[2-4句：是什么 + 为什么重要 + 点评]

[编号]. [标题]
[2-4句]

...（共 6-10 条）

━━━━━━━━━━━━━━━━━━
📖 今日深度解读
━━━━━━━━━━━━━━━━━━

今天深度拆解了 N 篇：
• [标题] — [一句话说这篇讲了什么]
  全文 → https://dongdongbear.github.io/Lighthouse/ai-research/news/${TODAY}/[slug]/

• [标题] — [一句话]
  全文 → https://dongdongbear.github.io/Lighthouse/ai-research/news/${TODAY}/[slug]/

━━━━━━━━━━━━━━━━━━
💡 今日推荐
━━━━━━━━━━━━━━━━━━

最值得深读：[标题]（[一句话理由]）
最值得动手试：[标题]（[一句话理由]）

完整日报 → https://dongdongbear.github.io/Lighthouse/ai-research/news/${TODAY}/daily/
```

## 第五步：输出

将推送文本写入 `/tmp/lighthouse-broadcast-${TODAY}.txt`

然后将推送文本作为最终输出直接打印（不要包含 DONE 标记，因为这段文字会直接用于飞书发送）。

## 自检清单（输出前确认）

- [ ] 三大厂动态是否一条不漏？
- [ ] 总条目 8-12 条？
- [ ] 每条都有实质内容，不是空洞的一句话？
- [ ] 深度解读的链接 slug 是否与实际文件名一致？
- [ ] 语气自然，不像机器人？
- [ ] 完整日报链接正确？
