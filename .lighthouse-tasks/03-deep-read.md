## 第三步：写入文件

每篇解读单独一个文件：
```
/tmp/Lighthouse/src/content/docs/ai-research/news/${TODAY}/<slug>.md
```

slug 规则：全小写，短横线分隔，简洁有辨识度（如 `openai-agent-monitoring`、`voxtral-tts-mistral`）。

frontmatter 必须包含 title 和 description 字段。

---

## 第四步：完善 daily.md

写完解读后检查 daily.md：

1. **总条目数**：目标 25-40 条，不足则补采
2. **结构完整**：frontmatter → 上期追踪回应 → 各区块 → 下期追踪问题
3. **下期追踪问题**（精髓！这是日报连续性的灵魂）：
   - 在文件末尾添加恰好 3 个问题
   - 问题必须**具体、可追踪、有时效性**
   - 好的问题：「DeepSeek R2 是否会在本周内正式发布？」「Anthropic vs 五角大楼禁令上诉结果？」
   - 坏的问题：「AI 未来会怎样？」「哪些公司会成功？」（太泛、无法追踪）
   - 优先从今日 ⭐ 标记的重大事件中提问——它们明天最可能有后续
   - 格式：
   ```markdown
   ---

   ## 下期追踪问题

   1. [具体事件] + [具体想知道的后续]？
   2. [具体事件] + [具体想知道的后续]？
   3. [具体事件] + [具体想知道的后续]？
   ```
4. **链接检查**：所有信源链接 https:// 开头
5. **更新 title**：frontmatter 的 title 更新为今日最大亮点

## 第五步：部署

```bash
cd /tmp/Lighthouse && \
  git add -A && \
  git commit -m "deep-read ${TODAY}: [解读了哪几篇的简称]" && \
  git push && \
  cd astro-app && npx astro build && npx pagefind --site dist && \
  cp -r dist/* /usr/share/nginx/html/Lighthouse/
```

故障处理：
- build 失败 → 检查 frontmatter 格式（title/description 必须有，特殊字符需要引号）
- git push 失败 → `git pull --rebase && git push`
- pagefind 失败 → 检查 dist 目录是否存在
