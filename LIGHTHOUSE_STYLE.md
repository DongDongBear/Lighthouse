# LightHouse 设计规范（当前版本）

## 品牌与定位
- 名称：**LightHouse**
- 核心语气：工程理性 + AI 时代方法论
- 文案关键词：模型协作 / 系统设计 / 持续迭代

## 视觉风格
- 主风格：黑白灰、极简、克制
- 交互原则：轻反馈，不花哨
- 卡片 hover：仅上浮（`translateY(-4px)`），不改变边框粗细

## 首页结构
1. Intro（LightHouse + BellLab/AI 定位）
2. 模块标题（AI 语境）
3. 两张核心卡片（Unity / Electron）

## 卡片规范
- 媒体区：
  - 背景：`rgba(0,0,0,0.02)`，Dark：`rgba(255,255,255,0.05)`
  - 高度：`320px`
  - padding：`24px`
  - 圆角：`0`
- 标题：14px / medium
- 描述：14px / light / relaxed

## 布局规范
- 首页：大屏固定左右留白（200px）
- 文档页：恢复正常阅读留白，不使用 200px 固定值
- 内容区与左右侧边栏保持可阅读间距，避免“贴脸”

## 禁止项
- 首页标题蓝色渐变
- 标题 hover 出现 `#` 锚点符号
- hover 导致边框变粗/变黑突兀

## 发布检查（每次改样式后）
1. `npm run docs:build` 必须通过
2. 检查首页：卡片可点击、图片可加载、无源码文本泄漏
3. 检查文档页：正文宽度正常、侧栏间距正常
4. 检查移动端：卡片单列、无溢出
