# 02. 强化学习与后训练：从 RLHF 到 OAPL

这是理解 KARL 最关键的一章。KARL 的核心贡献就是用 RL 后训练把通用 LLM 变成专业搜索 Agent。

---

## 一、RL 的基本框架

强化学习的核心循环：

```
Agent（模型）
  ↓ 产生行为
Environment（环境）
  ↓ 返回奖励 + 新状态
Agent 根据奖励调整行为策略
```

对于 LLM：

- **Agent** = 模型本身
- **行为** = 生成一段 token 序列（一条 rollout / trajectory）
- **环境** = 任务评测系统（比如检查答案正确性）
- **奖励** = 任务分数

### 核心术语

**Policy（策略）π(y|x)**

策略就是模型。给定输入 x，它定义了所有可能输出 y 的概率分布。

训练的目标就是找到一个更好的策略——让高分输出的概率更高。

**Reward（奖励）r(x, y)**

环境对模型输出的打分。在 KARL 中：

- BrowseComp-Plus：找到正确实体得 1 分，否则 0 分
- TREC-Biogen：nugget completeness 分数，连续值 [0, 1]

**Rollout（轨迹）**

模型的一次完整尝试。在 Agent 场景中包含：

```
[思考] → [搜索查询 1] → [检索结果 1] → [思考] → [搜索查询 2] → ... → [最终答案]
```

一条 rollout 可能有 5 步（简单问题）到 200+ 步（BrowseComp-Plus 的复杂问题）。

**Episode / Group**

同一个问题的多条 rollout 构成一个 group。KARL 中每个问题通常跑 8 条 rollout，然后对比它们的奖励来计算学习信号。

---

## 二、优势函数：学习信号的核心

RL 的关键问题：如果一条 rollout 得了 0.6 分，我该鼓励还是抑制这个行为？

答案取决于**相对表现**，而不是绝对分数。

### Advantage（优势）

```
A(x, y) = r(x, y) - V(x)
```

- r(x, y)：这条 rollout 的分数
- V(x)：这个问题的"期望分数"（所有 rollout 的加权平均）

直觉：

- A > 0：这条 rollout 比平均好 → 鼓励
- A < 0：这条 rollout 比平均差 → 抑制

### 为什么不直接用 reward？

因为不同问题的难度不同。一个简单题得 0.6 分可能很差（平均都 0.9），一个难题得 0.6 分可能很好（平均只有 0.2）。

优势函数自动归一化了难度差异。

---

## 三、KL 正则化：不跑偏的约束

如果只最大化 reward，模型可能走极端——比如学会一种"作弊式"的捷径，总是输出某种特定格式骗到高分，但实际能力退化。

**KL 散度**衡量两个概率分布的差异：

```
KL(π || π_ref) = E_{y~π} [log(π(y|x) / π_ref(y|x))]
```

直觉：新策略 π 和参考策略 π_ref 的"距离"。KL = 0 意味着完全一样。

RL 的标准目标变成：

```
max E[r(x, y)] - β · KL(π || π_ref)
```

**β 是"安全绳"强度**：
- β 大：约束强，模型不敢偏离太远，训练稳定但提升幅度有限
- β 小：约束弱，模型可以大幅调整行为，但可能不稳定

KARL 的 OAPL 使用两个独立的 β（β₁ 用于价值估计，β₂ 用于 KL 约束），提供更灵活的调节。

---

## 四、On-policy vs Off-policy

这是理解 KARL 为什么选 OAPL 而不是 GRPO 的关键区分。

### On-policy（在线策略）

训练数据来自"当前最新的模型"。

```
循环：
  1. 用当前模型 π_t 采样一批 rollout
  2. 计算奖励和优势
  3. 更新模型 → π_{t+1}
  4. 刚采样的数据作废，回到步骤 1
```

代表算法：PPO、GRPO。

优点：数据分布和当前策略一致（无分布偏移）
缺点：
- 每轮训练都要重新采样（对 Agent 场景极其昂贵，每条 rollout 涉及大量工具调用）
- 推理和训练必须同时进行，系统复杂度高
- 超参搜索时每组超参都要重新采样

### Off-policy（离线策略）

训练数据来自"之前版本的模型"，可以反复使用。

```
1. 用模型 π_old 一次性采样大量 rollout（比如 10 万条）
2. 在这批数据上多次训练更新 → π_new
3. 如需继续，用 π_new 重新采样
```

代表算法：OAPL。

优点：
- 数据采集成本分摊（一批数据跑多次训练）
- 推理和训练完全解耦（可以分别优化）
- 超参搜索只需采样一次数据
- 对 MoE 模型更稳定（不需要 router replay）

缺点：
- 存在分布偏移（训练数据来自旧策略）
- 需要算法层面处理这个偏移

---

## 五、GRPO 的做法（KARL 的对比对象）

GRPO（Group Relative Policy Optimization）是目前最流行的 LLM RL 算法之一（DeepSeek 系列使用）。

### 核心流程

```
对每个 prompt x：
  1. 用当前策略 π_θ 采样 G 条 rollout {y_1, ..., y_G}
  2. 计算每条的奖励 r_i
  3. 组内归一化得到优势：Â_i = (r_i - mean(r)) / std(r)
  4. 更新策略：提高高优势 rollout 的概率，降低低优势的
```

### GRPO 在 Agent/MoE 场景的问题

1. **On-policy 采样成本**：Agent 的 rollout 涉及大量工具调用（KARL 中一条 BrowseComp-Plus 轨迹可能 200 步），每轮训练都重新采样代价极高
2. **MoE Router 不稳定**：在线更新时 Router 和 Expert 的梯度互相干扰
3. **需要额外 trick**：clipped importance weighting（裁剪重要性权重）、off-policy 数据删除、router replay 等

---

## 六、OAPL 的做法（KARL 的核心算法）

OAPL（Optimal Advantage-based Policy Optimization with Lagged Inference policy）是 KARL 的关键创新。

### 数学直觉

从 KL 正则化 RL 目标出发，存在一个理论最优策略：

```
π*(y|x) ∝ π_ref(y|x) · exp(r(x,y) / β)
```

直觉：最优策略应该在参考策略的基础上，按奖励的指数函数重新加权——高奖励的输出概率提升，低奖励的下降。

OAPL 的目标是让当前策略"逼近"这个理论最优，通过最小二乘回归：

```
Loss = Σ (β · log(π(y|x) / π_ref(y|x)) - (r(x,y) - V̂*(x)))²
```

左边：当前策略相对参考策略的"偏好变化量"
右边：根据奖励应该变化多少（优势）

V̂\*(x) 用 group rollouts 估计（不需要单独训练 value model）：

```
V̂*(x) = β · log(1/G · Σ_i exp(r(x, y_i) / β))
```

这是一个 log-sum-exp 形式，比简单平均更关注高奖励 rollout。

### 为什么 OAPL 更适合 KARL 的场景

| 特性 | GRPO | OAPL |
|------|------|------|
| 采样策略 | 每轮重新采样（on-policy） | 一次大批采样，多次训练（off-policy） |
| MoE 稳定性 | 需要 trick | 天然稳定 |
| 超参搜索成本 | 每组超参重新采样 | 同一批数据跑多组 |
| 系统复杂度 | 推理 + 训练耦合 | 推理 + 训练解耦 |
| 分布偏移处理 | clipping + 数据删除 | 在优化目标中天然处理 |

## 七、多步 Agent 场景的 RL 特殊处理

Agent 的 rollout 不是一段纯文本，而是包含交替的"模型生成"和"工具返回"：

```
[模型：搜索查询] → [工具：返回文档 A, B, C] → [模型：分析 + 新查询] → [工具：返回文档 D, E] → [模型：最终答案]
```

### Token 掩码

RL 训练只更新模型对**自己生成的 token** 的概率。工具返回的 token 不是模型"说"的，不应该参与损失计算。

```
[模型生成 → 计算损失] [工具返回 → 掩码] [模型生成 → 计算损失] [工具返回 → 掩码] ...
```

### 轨迹分段

BrowseComp-Plus 的长轨迹可能有 200 步 + 多次压缩。把整条轨迹放进 GPU 训练不现实。

KARL 的做法：在每个**压缩步骤**处切分轨迹。每段独立训练，但使用**整条轨迹的最终奖励**作为信号。

```
段 1: [初始 prompt → 搜索 1-50 → 压缩]  ← 奖励 = 最终答案得分
段 2: [压缩摘要 → 搜索 51-100 → 压缩]   ← 奖励 = 最终答案得分
段 3: [压缩摘要 → 搜索 101-150 → 答案]   ← 奖励 = 最终答案得分
```

### 压缩步骤也纳入 RL

压缩步骤本身也创建训练样本：x = 待压缩的历史，y = 模型生成的摘要。同样用最终奖励作为信号。

这意味着模型会学到**为了最终答对而压缩**——保留对后续搜索有用的信息，丢弃无关的。这比通用摘要模型更强。

## 八、迭代自举训练

KARL 不是训一次就结束，而是迭代进行：

```
GLM 4.5 Air → 合成数据 → OAPL Iter.1 → KARL v1
KARL v1     → 合成更好的数据 → OAPL Iter.2 → KARL v2
KARL v2     → 合成更难的数据 → OAPL Iter.3 → KARL v3
```

每轮迭代：
- 模型更强 → 合成的问题更难、质量更高
- 通过率过滤的阈值也提高（防止训练数据太简单）
- 3 轮都有持续提升，没到平台期

---

## 九、多任务 RL vs 蒸馏

KARL 在两个任务上训练：BrowseComp-Plus（深搜：反复缩小范围找唯一实体）和 TREC-Biogen（广搜：大范围收集信息写报告）。

替代方案是"多专家蒸馏"：分别训两个专家，再用 SFT 把它们合并。

关键实验结果：

| 方法 | 分布内 | 分布外 | 总分 |
|------|--------|--------|------|
| 蒸馏 | 69.1 | 59.4 | 64.7 |
| 蒸馏 + TTC | 75.3 | **59.6**（几乎没涨） | 64.8 |
| KARL RL | 69.4 | 53.7 | 58.9 |
| KARL RL + TTC | 78.4 | **62.7**（+9.0） | 67.9 |

蒸馏在分布外几乎不涨——因为它教的是"模仿专家行为"，模仿只在见过的任务上有效。

RL 在分布外大幅上涨——因为它学的是"通用搜索策略"，这种策略可以迁移。

---

## 十、RL 不只是 Sharpening

学界有一个争论：RL 后训练到底是学了新能力，还是只是"锐化"了基座模型已有的分布（让已经会做的题更一致地做对）？

KARL 用 **max@k 分析** 回答了这个问题：

- max@k = 采样 k 次，取最好的一次。它衡量的是"模型在最好情况下能做到什么"——即能力边界

如果 RL 只是 sharpening，那 max@k 在 k 很大时不会变（因为能力边界没扩展）。

KARL 的结果：**max@k 在所有 k 值上都提升了**。

- 训练后 max@1 ≈ 基座的 max@8
- 训练后 max@2 已经超过基座的 max@16

意思是：**KARL 2 次就能解决基座 16 次都解不了的问题。** 这证明 RL 真正扩展了能力边界。
