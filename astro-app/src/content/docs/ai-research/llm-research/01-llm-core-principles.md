# 01. LLM 核心原理：从 Transformer 到训练全流程

---

## 一、LLM 到底在做什么

大语言模型的核心任务只有一个：**给定前面的 token 序列，预测下一个 token 的概率分布**。

```
P(x_t | x_1, x_2, ..., x_{t-1})
```

Token 是模型的最小文本单位。中文大约一字一个 token，英文会把长词拆成子词片段（"unhappiness" → "un" + "happi" + "ness"）。模型的词汇表通常 32K-150K 个 token。

生成时是自回归（autoregressive）的：预测一个 token，拼到输入末尾，再预测下一个，循环往复。ChatGPT 回答一个问题，可能循环了上千次。

看似简单，但当参数量达到千亿级、训练数据达到万亿 token 级，这个"下一个 token 预测"任务就涌现出了推理、编程、翻译、创作等能力。这是目前学界仍在研究的涌现现象（emergent ability）。

## 二、Transformer 架构

几乎所有现代 LLM 都基于 2017 年 Google 提出的 Transformer。理解它需要抓住三层结构。

### 2.1 Embedding 层：从离散到连续

Token 是离散符号（词表里的一个编号），但神经网络需要连续数值输入。

**Embedding** 就是一张查找表：每个 token ID 对应一个高维向量（常见 4096 维或更高）。

```
token "猫" → ID 8372 → 向量 [0.12, -0.34, 0.56, ..., 0.78]  (4096 维)
token "狗" → ID 5201 → 向量 [0.15, -0.31, 0.61, ..., 0.72]  (4096 维)
```

语义相近的 token，向量在高维空间中距离更近。这不是手动设定的，而是训练过程中自然学到的。

还需要加上**位置编码**（Positional Encoding），因为 Transformer 本身不感知顺序——你要告诉它"这是第 1 个词""这是第 200 个词"。现代做法（RoPE 等）会把位置信息编码进注意力计算中。

### 2.2 Self-Attention：核心中的核心

注意力机制回答的问题是：**生成当前 token 时，应该"关注"前面哪些 token，关注多少？**

#### 直觉

读这句话：

> "小明昨天去了北京，**他**在那里参观了故宫。"

读到"他"时，你大脑自动关注"小明"——因为"他"指代"小明"。这就是注意力。

#### 计算过程

每个 token 的向量被线性变换成三个角色：

- **Query (Q)**：我在找什么信息？
- **Key (K)**：我有什么信息可以提供？
- **Value (V)**：如果被选中，我提供的具体内容是什么

注意力分数 = Q 和 K 的点积（衡量相关性），归一化后作为权重，对 V 加权求和。

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) · V
```

`sqrt(d_k)` 是缩放因子，防止点积值过大导致 softmax 饱和。

#### Multi-Head Attention

不是只做一次注意力，而是并行做 h 次（h 通常是 32、64、128），每次用不同的 Q/K/V 投影矩阵。

每个"头"学到不同的关注模式：有的头关注语法关系，有的关注语义引用，有的关注位置距离。最后把所有头的输出拼接起来。

#### 因果掩码（Causal Mask）

GPT 类模型（decoder-only）在训练和推理时，每个位置只能看到它**前面**的 token，不能看后面的。这通过一个上三角掩码矩阵实现，把"未来位置"的注意力分数设为负无穷。

这就是为什么 LLM 是"从左到右"生成的。

#### 计算复杂度

对于长度 n 的序列，注意力需要计算 n x n 的分数矩阵，**复杂度是 O(n^2)**。这就是上下文窗口有长度限制的根本原因。128K token 的窗口意味着一个 128K x 128K 的注意力矩阵（约 160 亿个元素）。

实际中通过 Flash Attention、稀疏注意力等工程优化来降低实际开销。

### 2.3 FFN（前馈网络）：知识存储

每层 Transformer 除了注意力，还有一个前馈网络（Feed-Forward Network）：

```
FFN(x) = W_2 · activation(W_1 · x + b_1) + b_2
```

通常 FFN 的隐藏层维度是 embedding 维度的 4 倍（比如 4096 → 16384 → 4096）。

研究表明：**模型的大部分"知识"存储在 FFN 的权重矩阵中**，而注意力层更多负责"信息路由"——决定哪些知识在当前上下文中被调用。

### 2.4 层堆叠

一个完整的 Transformer 层 = Multi-Head Attention + FFN + 残差连接 + LayerNorm。

现代 LLM 通常堆叠 32-128 层：

```
输入 tokens → Embedding → [Attn + FFN] x N 层 → 输出概率分布
```

浅层学到词法和语法模式，深层学到语义推理和世界知识。GPT-3 有 96 层，总参数 1750 亿。

## 三、MoE：以小博大的工程方案

### 为什么需要 MoE

参数越多，模型潜力越大。但推理时如果每个 token 都要过全部参数，计算成本线性增长。

**MoE（Mixture of Experts）** 的解法：把 FFN 拆成多个"专家"，每个 token 只激活其中 2-8 个。

```
Dense Model（每 token 全部参数参与）:
  token → 全部 1750 亿参数 → 输出

MoE Model（每 token 只用部分参数）:
  token → Router 网络 → 选 Expert 3 和 Expert 11
                       → 只过这两个 Expert 的参数 → 输出
```

DeepSeek-V3：总参数 6710 亿，每 token 只激活 370 亿。效果接近同等总参数量的 Dense 模型，但计算成本只有 1/18。

### Router 的训练难题

Router 网络决定"哪个 token 去哪个 Expert"，它的梯度更新和 Expert 本身的梯度更新互相耦合，容易导致：

- **负载不均**：某些 Expert 被大量选中，其他 Expert 饥饿
- **训练不稳定**：梯度信号在 Router 和 Expert 之间振荡

这就是 KARL 论文中提到的"GRPO 在 MoE 上需要 router replay 等 trick"的背景。KARL 的 OAPL 方法因为是离线训练，天然规避了这个问题。

## 四、训练三阶段

### 4.1 预训练（Pretraining）：打地基

**任务**：下一个 token 预测（Next Token Prediction）。

**数据**：互联网文本、书籍、代码、论文——万亿 token 级别。

**过程**：

```
对于训练集中每个文档的每个位置 t：
  1. 把 x_1, ..., x_{t-1} 喂给模型
  2. 模型预测 P(x_t | x_1, ..., x_{t-1})
  3. 计算交叉熵损失 L = -log P(x_t)
  4. 反向传播更新参数
```

交叉熵损失的直觉：如果模型给正确 token 分配了很高的概率，损失就小；如果给了很低的概率，损失就大。训练就是不断降低这个损失。

**资源消耗**：GPT-4 级别的预训练需要数万张 A100/H100 GPU，训练数月，成本上亿美元。

**结果**：模型获得了广泛的语言能力和世界知识，但它的行为是"像互联网文本一样续写"，不一定听话、安全、有用。

### 4.2 监督微调（SFT）：学会交互格式

**任务**：在高质量的"指令-回答"数据对上继续训练。

```
输入：请用一句话解释量子纠缠。
输出：两个粒子一旦纠缠，无论相距多远，测量一个会瞬间影响另一个的状态。
```

SFT 的数据量远小于预训练（通常数万到数十万条），但质量极高——通常由人类标注员或强模型（如 GPT-4）生成。

**结果**：模型从"续写机器"变成"会对话的助手"。但 SFT 更多是教格式和风格，不太能教会复杂的决策能力。

### 4.3 后训练 / 强化学习（Post-training / RL）：强化关键能力

这一步有两种主要目的：

#### (a) 对齐（Alignment）

RLHF（Reinforcement Learning from Human Feedback）的经典流程：

1. 让模型对同一问题生成多个回答
2. 人类标注员排序（哪个更好）
3. 训练一个 Reward Model 拟合人类偏好
4. 用 RL（如 PPO）优化模型，使其生成更符合 Reward Model 偏好的回答

这一步让 ChatGPT 从"有能力但不可控"变成"有能力且安全可用"。

#### (b) 能力强化

**这正是 KARL 所在的领域。**

不是用人类偏好打分，而是用**任务完成度**打分：

- 搜索任务：找到正确答案了吗？找了多少个 nugget？
- 数学任务：最终答案对不对？
- 编程任务：代码能不能通过测试？

KARL 用 RL 把一个通用 LLM（GLM 4.5 Air）训练成专业的知识搜索 Agent。训练信号不来自人类偏好，而来自搜索结果的正确性。

## 五、生成策略：温度、采样、Beam Search

模型输出的是概率分布，怎么从分布中选 token 决定了生成质量。

### 温度（Temperature）

```
P'(x) = softmax(logits / T)
```

- T = 0（或极小）：几乎总选概率最高的 token（贪心），确定性强
- T = 0.7：适度随机
- T = 1.0+：高随机性

### Top-k 和 Top-p

- **Top-k**：只从概率最高的 k 个 token 中采样
- **Top-p（nucleus sampling）**：只从累积概率达到 p 的 token 集中采样

这些策略在 KARL 的推理时计算扩展（TTC）中至关重要——同一个问题用不同随机种子采样多次，就能得到不同的搜索轨迹。

## 六、上下文窗口：模型的工作记忆

上下文窗口（Context Window）是模型一次能处理的最大 token 数。

```
GPT-3:      4K token
GPT-4:      128K token
Claude 4.6: 200K token
GPT-5.4:    1M token
```

**窗口外的信息模型看不到。** 对于 Agent 场景（像 KARL），每次检索都往上下文里塞文档，几十步后窗口就满了。这就是为什么 KARL 需要上下文压缩——让模型自己总结历史，腾出空间继续搜索。

**KV Cache**：推理时，已计算过的 token 的 Key/Value 向量会缓存起来，新 token 只需要计算自己的 Q 并和所有已有 K/V 做注意力。这把每步推理的复杂度从 O(n^2) 降到 O(n)，但缓存本身占大量显存。

---

## 七、参数知识 vs 外部知识

模型参数中的知识有三个根本局限：

1. **时效性**：训练数据有截止日期
2. **私有性**：企业数据、未公开文档不在训练集中
3. **可靠性**：模型可能幻觉（hallucination）——生成看似合理但错误的内容

这三个局限催生了 RAG（检索增强生成）和 Agent 范式——让模型**主动访问外部数据**，而不仅仅依赖参数记忆。KARL 论文把这种需要外部知识支撑的推理称为 **Grounded Reasoning**。
